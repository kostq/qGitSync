# qGitSync

**Синхронизация папок между компьютерами через GitHub.**

[English version](README.md)

## Скачать

Свежая версия — на **[странице релизов](https://github.com/kostq/qGitSync/releases/latest)**:

- **`qGitSync-<версия>-setup.exe`** (рекомендуется) — установщик: выбор папки,
  ярлыки в меню «Пуск» и на рабочем столе, автозапуск по желанию, корректное
  удаление. Права администратора не нужны.
- **`qGitSync-<версия>-portable.exe`** — один файл, работает откуда угодно
  (хоть с флешки), без установки.

Обе версии хранят настройки в `%LOCALAPPDATA%\qGitSync`, между ними можно
свободно переключаться; удаление программы не трогает ваши папки и настройки.

qGitSync следит за вашими папками и синхронизирует их с репозиториями GitHub.
Изменили файл — он автоматически закоммитится и отправится; открыли программу
на другом компьютере — изменения приехали.

## Возможности

- **Несколько папок**, каждая привязана к своему репозиторию GitHub,
  со своими настройками и журналом.
- **Три режима** (комбинируются): автоматически через N секунд после
  последнего изменения, по таймеру каждые N минут, вручную по кнопке.
- **Быстрая доставка**: лёгкая проверка спрашивает GitHub каждые N минут
  (по умолчанию 1) и запускает полный синк только если там реально есть
  новое — изменения с другого компьютера приезжают за минуту-две.
- **Безопасные конфликты**: если один файл изменён на двух компьютерах,
  синхронизация не останавливается — версия с сервера остаётся под исходным
  именем, локальная сохраняется рядом как `файл (conflict ИМЯ-ПК дата).расш`,
  и обе разъезжаются по всем компьютерам. Ничего не теряется молча.
- **Управление репозиториями**: список, создание и удаление репозиториев
  GitHub прямо из программы (токен доступа хранится в защищённом хранилище
  Windows, а не в файлах).
- **Статусы**: 🟢 синхронизировано / 🔄 идёт / 🔴 ошибка / ⚠️ конфликт,
  счётчики изменений, уведомления из трея при проблемах.
- **Установка с нуля**: один `qGitSync.exe`. Если Git не найден, программа
  предложит скачать портативную копию (~60 МБ) только для себя.
- **Трей**: закрытие окна не останавливает синхронизацию; автозапуск
  с Windows. Повторный запуск просто показывает уже работающее окно —
  дубликаты процессов не плодятся.
- **Языки**: английский (по умолчанию), русский, испанский.

## Быстрый старт

1. Запустите `qGitSync.exe` (или `py app.py` из исходников).
2. **Токен GitHub** (один раз): панель → *Токен GitHub* → *Открыть страницу
   токенов* → создайте classic-токен с правами `repo` + `delete_repo` →
   вставьте → *Проверить и сохранить*.
3. **Добавить папку**: выберите папку и способ привязки — создать новый
   репозиторий прямо из программы, выбрать свой из списка, вставить ссылку
   или пока оставить локально.
4. Готово — первая синхронизация запустится сама.

**Второй компьютер:** запустите qGitSync, сохраните тот же токен,
*Добавить папку* → укажите **пустую** папку → *Выбрать из моих
репозиториев* → тот же реп. Файлы склонируются и будут синхронизироваться
в обе стороны.

## Как это работает

Каждая синхронизация — это `git add -A` → `commit` →
`pull --rebase --autostash` → `push`. Если один и тот же файл изменён на двух
компьютерах одновременно, qGitSync сохраняет обе версии: пришедшая остаётся
под исходным именем, локальная сохраняется конфликт-копией рядом, и обе
попадают на все компьютеры. Полная история синхронизаций хранится на GitHub —
любое прошлое состояние можно восстановить.

Настройки: `%LOCALAPPDATA%\qGitSync\config.json`; портативный Git (если
скачивался): `%LOCALAPPDATA%\qGitSync\MinGit`.

## Ограничения GitHub

- Файлы больше 100 МБ GitHub не принимает (для них нужен Git LFS).
- Репозиторий лучше держать в пределах ~1–2 ГБ.
- Отлично подходит для документов, кода и заметок; не подходит для фильмов
  и бэкапов дисков.

## Сборка из исходников

```
py -m pip install -r requirements.txt
py app.py                # запуск

py -m pip install pyinstaller

# портативный один файл → dist\qGitSync.exe
py -m PyInstaller --noconfirm --onefile --windowed --name qGitSync --icon icon.ico --add-data "icon.ico;." app.py

# установщик (нужен Inno Setup 6) → dist\qGitSync-setup.exe
py -m PyInstaller --noconfirm --windowed --name qGitSync --icon icon.ico --add-data "icon.ico;." --distpath build_installer\dist --workpath build_installer\work app.py
iscc installer.iss
```
